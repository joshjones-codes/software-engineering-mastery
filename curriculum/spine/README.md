# Spine — The Build Track

The primary track. Eight stages, each driven by a real product bottleneck. One module ≈ one weekend loop (Read → Build → Integrate → Reflect → Teach). Work top to bottom, but a stage doesn't need to be 100% complete before starting the next — `must` modules gate progress, `should`/`nice` can be back-filled.

| Stage | Modules | Focus |
|---|---|---|
| [Production Backend](01-production-backend/README.md) | 13 | Turn CRUD into production software. |
| [Databases](02-databases/README.md) | 9 | Go from 'uses Postgres' to understanding what the database is doing and why. |
| [Caching](03-caching/README.md) | 4 | The first real scaling tool — and the first real consistency problem. |
| [Async Systems](04-async-systems/README.md) | 9 | The biggest conceptual leap: work that happens later, elsewhere, and sometimes twice. |
| [Distributed Systems](05-distributed-systems/README.md) | 12 | Staff-level thinking: what happens when there is more than one machine and any of them can fail. |
| [Storage, Search & Analytics](06-storage-search-analytics/README.md) | 7 | Files, documents, and turning operational data into questions you can answer. |
| [Infrastructure & Operations](07-infrastructure/README.md) | 11 | Own the machine your code runs on and the path it takes to get there. |
| [AI Systems](08-ai-systems/README.md) | 8 | Applied AI engineering on top of everything above — agents and RAG are distributed systems with a model in the loop. |
